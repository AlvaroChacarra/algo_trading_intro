Bloque 2 — Los errores son pistas

Rompe código a propósito

Un error no es un castigo — es Python diciéndote exactamente dónde y por qué no puede continuar. Aprender a leer errores es la habilidad más útil de un programador.

🔧 Reto 1: Paréntesis perdido Sin resolver

Python no entiende la frase — le falta una pieza de sintaxis. Arréglalo.

SyntaxError — Python no entiende la estructura. Cuenta los paréntesis: ¿cuántos abren y cuántos cierran?
🔧 Reto 2: Variable fantasma Sin resolver

La sintaxis es correcta, pero Python no encuentra algo que necesita. ¿Qué falta?

NameError — La frase se entiende, pero ask no existe. Necesitas definirla antes de usarla.
🔧 Reto 3: Tipos incompatibles Sin resolver

Todo existe y la sintaxis es válida, pero la operación no tiene sentido. ¿Por qué falla?

TypeError — price es texto ("100000"), no un número. Quita las comillas para que sea un int.
🔧 Reto 4: División imposible Sin resolver

Todo es correcto... hasta que los números no cuadran. ¿Cómo lo arreglas?

ZeroDivisionError — total es 0 porque buy y sell son iguales. Necesitas verificar antes de dividir, o cambiar la fórmula.

Python no entiende la frase

SyntaxError — falta un paréntesis, una comilla, dos puntos... Python ni siquiera llega a ejecutar.

Python entiende pero no puede hacerlo

NameError, TypeError, ZeroDivisionError — la frase está bien escrita, pero algo falla al ejecutarla.
