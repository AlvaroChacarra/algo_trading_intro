# Lesson 02 - Python OOP and Libraries for Trading

> Use code written by others. Then organize your own.

## Mission
Understand what a library is, what `import` really does, and why classes help when trading code starts to grow.

## Core ideas

- a library is code someone else already wrote
- `import` brings that code into your workspace
- `pandas` helps with tabular data fast
- a class groups data and behavior
- an object is one concrete instance of that class
- methods make logic reusable and easier to read

## Trading angle

- inspect a small table of orders and trades with `pandas`
- move from dictionaries to an `Order` object
- create a `Trade` object
- track `cash`, `position` and `equity`
- build a tiny reusable trading system

## End state

By the end, the student should be able to:

- explain what `import pandas as pd` means
- inspect a simple `DataFrame`
- define a class with `__init__`
- create objects with attributes and methods
- update a toy trading state with OOP
