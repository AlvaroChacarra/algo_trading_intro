El Problema: El Caos del Código "Espagueti"

Cuando empezamos, es normal escribir código así. Funciona, ¿pero a qué coste?

Un ejemplo de nuestro mini-juego:

# --- Héroe ---
heroe_nombre = "Aragorn"
heroe_vida = 100
heroe_ataque = 15

# --- Enemigo ---
orco_nombre = "Uglúk"
orco_vida = 80
orco_ataque = 10

# --- Lógica de combate ---
def atacar_al_orco(vida_orco, ataque_heroe):
    vida_orco -= ataque_heroe
    print(f"{heroe_nombre} ataca a {orco_nombre}!")
    print(f"{orco_nombre} ahora tiene {vida_orco} de vida.")
    return vida_orco

def atacar_al_heroe(vida_heroe, ataque_orco):
    # ... código similar ...
    pass

# Simulación
orco_vida = atacar_al_orco(orco_vida, heroe_ataque)
# ... y así sucesivamente ...

⚠️ ¡Esto se complica rápido!

  • ?
    ¿Y si queremos añadir otro orco?
    Tendríamos que crear `orco2_nombre`, `orco2_vida`... ¡Un lío!
  • ?
    ¿Y un mago con maná?
    Más variables sueltas (`heroe_mana`, `orco_resistencia_magica`...).
  • ?
    ¿Cómo sabemos qué datos pertenecen a quién?
    Los datos (`vida`) y las acciones (`atacar`) están separados. Es fácil cometer errores.

Necesitamos una forma de agrupar datos y comportamientos en "cajas" lógicas.

En el mundo real, un objeto como un coche no es solo una colección de datos sueltos. Es una entidad única que agrupa sus propias características y sus propios comportamientos.

La Analogía Clave: El Plano y la Casa

Esta es la diferencia fundamental entre una Clase y un Objeto.

La Clase: El Plano

Es la plantilla o la receta. No es algo real, sino el conjunto de instrucciones que define cómo será algo.

  • Define las características (atributos).
  • Define los comportamientos (métodos).
  • No se puede "usar" directamente, solo sirve para construir.

El Objeto: La Casa

Es la instancia real creada a partir de la clase. Es el resultado tangible de seguir las instrucciones del plano.

  • Tiene sus propios valores para los atributos.
  • Puede ejecutar los comportamientos definidos.
  • Con este sí interactuamos en nuestro programa.

Podemos construir muchos objetos (casas) a partir de una sola clase (plano).

Anatomía de una Clase: El Constructor `__init__`

Este es el "ritual de construcción" que se ejecuta al crear un nuevo objeto.

Nuestro primer "plano" en Python:

# 1. Definimos el plano con la palabra clave "class"
class Coche:
    # 2. El método constructor, se llama automáticamente
    #    al crear un objeto. ¡Es un método "mágico"!
    def __init__(self, color_inicial, marca_inicial):
        
        # 3. Creamos los "atributos" o características
        #    y les asignamos los valores iniciales.
        print("¡Construyendo un coche nuevo!")
        self.color = color_inicial
        self.marca = marca_inicial
        self.velocidad = 0 # Todos los coches empiezan parados

1. `class Coche:`

Es la declaración. Le decimos a Python: "voy a crear una nueva plantilla llamada Coche". Por convención, los nombres de las clases empiezan con mayúscula.

2. `def __init__(...)`

Este es el constructor. Su misión es inicializar el objeto. Recibe los "ingredientes" necesarios para construirlo (`color_inicial`, `marca_inicial`).

3. `self.color = ...`

Aquí creamos una variable dentro del objeto, llamada `color`, y le guardamos el valor que nos pasaron. A estas variables internas las llamamos atributos. Hacemos lo mismo para `marca` y `velocidad`.

El misterioso `self`...

Es la forma que tiene el objeto de referirse a sí mismo. Piensa que es como decir "mi color" o "mi marca". Lo veremos en detalle en la siguiente diapositiva.

El Misterioso `self`: La Clave de Todo

¿Por qué todos los métodos empiezan con `self`? Vamos a desvelar el secreto.

Veamos el código en acción:

class Coche:
    def __init__(self, color, marca):
        # 'self' aquí se refiere al objeto
        # que se está creando en este momento.
        self.color = color
        self.marca = marca

    def mostrar_info(self):
        # 'self' aquí se refiere al objeto
        # específico que llamó al método.
        print(f"Soy un coche de marca {self.marca} y color {self.color}.")

# Creamos dos objetos (casas) diferentes
coche_de_ana = Coche("Rojo", "Toyota")
coche_de_juan = Coche("Azul", "Ford")

# Cada objeto usa 'self' para acceder a su PROPIA información
coche_de_ana.mostrar_info()
# SALIDA: Soy un coche de marca Toyota y color Rojo.

coche_de_juan.mostrar_info()
# SALIDA: Soy un coche de marca Ford y color Azul.

¿Qué hace Python "por debajo"?

Cuando ejecutas:
coche_de_ana.mostrar_info()

Python lo traduce automáticamente a:

Coche.mostrar_info(coche_de_ana)

¡Python pasa el propio objeto (`coche_de_ana`) como el primer argumento! A ese primer argumento, por convención, lo llamamos `self`.

`self` es el "pronombre YO" del objeto.

Permite que cada objeto trabaje con sus propios datos sin confundirse con los de otros objetos del mismo tipo.

Atributos: Las Características del Objeto

Son las variables que viven dentro de un objeto y guardan su estado individual.

Accediendo a los datos:

class Coche:
    def __init__(self, color, marca):
        self.color = color
        self.marca = marca
        self.velocidad = 0

# Creamos dos objetos
coche_de_ana = Coche("Rojo", "Toyota")
coche_de_juan = Coche("Azul", "Ford")

# Usamos la "sintaxis del punto" para LEER un atributo
print(f"El coche de Ana es de color {coche_de_ana.color}")
# SALIDA: El coche de Ana es de color Rojo

print(f"La marca del coche de Juan es {coche_de_juan.marca}")
# SALIDA: La marca del coche de Juan es Ford

# También podemos MODIFICAR un atributo
coche_de_ana.velocidad = 50
print(f"Velocidad del coche de Ana: {coche_de_ana.velocidad} km/h")
# SALIDA: Velocidad del coche de Ana: 50 km/h

Cada Objeto, Sus Propios Datos

Imagina que cada objeto es una "caja" con sus propias etiquetas. Al crear `coche_de_ana` y `coche_de_juan`, esto es lo que ocurre en la memoria:

Objeto: `coche_de_ana`

color: "Rojo"

marca: "Toyota"

velocidad: 0

Objeto: `coche_de_juan`

color: "Azul"

marca: "Ford"

velocidad: 0

Modificar `coche_de_ana.velocidad` NO afecta a `coche_de_juan.velocidad`.

Métodos: Los Comportamientos del Objeto

Son las funciones que definen lo que un objeto puede hacer, normalmente modificando sus propios atributos.

Añadiendo acciones a nuestro `Coche`:

class Coche:
    def __init__(self, color, marca):
        self.color = color
        self.marca = marca
        self.velocidad = 0 # Atributo de estado

    # Un método para cambiar el estado (velocidad)
    def acelerar(self, cantidad):
        self.velocidad += cantidad
        print(f"¡Brum, brum! Velocidad actual: {self.velocidad} km/h")

    # Otro método que usa los atributos
    def tocar_bocina(self):
        print(f"¡El {self.marca} {self.color} hace Beep, beep!")

# Creamos un coche
mi_ferrari = Coche("Rojo", "Ferrari")

# Llamamos a sus métodos usando la sintaxis del punto
mi_ferrari.tocar_bocina()
# SALIDA: ¡El Ferrari Rojo hace Beep, beep!

mi_ferrari.acelerar(50)
# SALIDA: ¡Brum, brum! Velocidad actual: 50 km/h

mi_ferrari.acelerar(30)
# SALIDA: ¡Brum, brum! Velocidad actual: 80 km/h

¿Cómo Funcionan?

Son Funciones "Especiales"

Un método es simplemente una función que "vive" dentro de una clase. La única diferencia es que siempre reciben `self` como primer parámetro.

Modifican el Estado Interno

La principal utilidad de un método es leer o cambiar los valores de los atributos del propio objeto. El método `acelerar` usa `self.velocidad` para saber la velocidad actual y poder aumentarla.

Atributos (Datos) + Métodos (Acciones) = Objeto Completo

¡Ahora nuestro objeto no solo tiene datos, sino que también puede realizar acciones con ellos! Está completo.

¡Dando Vida al Objeto! El Ciclo Completo

Este es el flujo de trabajo fundamental en OOP: Definir, Crear e Interactuar.

1 Definir el Plano (la Clase)

Escribimos el código que sirve como plantilla para todos nuestros futuros objetos `Coche`.

class Coche:
    def __init__(self, color, marca):
        self.color = color
        self.marca = marca
        self.velocidad = 0

    def acelerar(self, cantidad):
        self.velocidad += cantidad

    def mostrar_info(self):
        return f"Coche {self.marca} de color {self.color} (Vel: {self.velocidad} km/h)"

2 Construir el Coche (el Objeto)

Usamos la clase como si fuera una función para crear una instancia real y tangible.

# Creamos una instancia de la clase Coche
mi_coche = Coche("Plata", "Honda")

3 Usar el Coche (Interactuar)

Una vez creado, podemos llamar a sus métodos y leer/modificar sus atributos.

# Leemos un atributo
print(mi_coche.color)
# SALIDA: Plata

# Llamamos a un método para cambiar su estado
mi_coche.acelerar(100)

# Llamamos a otro método para ver la información actualizada
print(mi_coche.mostrar_info())
# SALIDA: Coche Honda de color Plata (Vel: 100 km/h)

Pilar 1: Encapsulación (El Coche es Tuyo)

La idea es que un objeto debe controlar su propio estado. ¡Nadie más debería "meter mano" donde no le llaman!

La Analogía del Conductor

Imagina que tu objeto `Coche` es tu coche real. Tú, como conductor, eres el único que debe tener control total sobre sus funciones internas a través de una interfaz clara (el salpicadero).

Un pasajero (otro trozo de código) no debería poder bajar el aire acondicionado directamente. Debería pedírtelo a ti, y tú (el objeto) decides si hacerlo o no a través de tus controles.

Interfaz Pública (El Salpicadero)

Son los métodos que ofreces al mundo exterior para interactuar contigo: `acelerar()`, `frenar()`, `subir_volumen_radio()`.

Estado Interno (La Mecánica)

Son los atributos que gestionas internamente: `velocidad`, `temperatura_ac`, `volumen_radio`.

Objeto `Coche`

-- Interfaz Pública --

acelerar(), frenar(), ajustar_ac()

-- Estado Interno Protegido --

_velocidad, _temperatura, _volumen

Encapsulación en Código: Seguridad

Usamos métodos como "guardianes" para proteger nuestros atributos de valores no deseados.

Protegiendo la velocidad:

class Coche:
    def __init__(self, color, marca):
        self.color = color
        self.marca = marca
        # Con un "_" indicamos que es un atributo "privado"
        self._velocidad = 0

    def acelerar(self, cantidad):
        # Podríamos añadir un límite máximo de velocidad aquí
        self._velocidad += cantidad

    def frenar(self, cantidad):
        # El método "guardián" protege el atributo
        if self._velocidad < cantidad:
            self._velocidad = 0 # No podemos tener velocidad negativa
        else:
            self._velocidad -= cantidad

    def get_velocidad(self):
        return self._velocidad

mi_coche = Coche("Rojo", "Ferrari")

# 🚫 MAL: Acceso directo y peligroso
mi_coche._velocidad = -50 # ¡Esto no tiene sentido!

# ✅ BIEN: Usamos el método que contiene la lógica de seguridad
mi_coche.frenar(100)
print(f"Velocidad segura: {mi_coche.get_velocidad()}")
# SALIDA: Velocidad segura: 0

El Método Guardián

El método `frenar` no se limita a restar. Primero comprueba si la operación tiene sentido. Actúa como un filtro, un "guardia de seguridad" que protege al atributo `_velocidad`.

Acceso Directo = Riesgo

Modificar `_velocidad` directamente nos permite asignarle valores ilógicos, creando bugs (errores) en nuestro programa.

Acceso por Método = Control

Llamar a `frenar()` nos garantiza que el estado del objeto siempre será coherente y válido.

Nota: En Python, el `_` (guion bajo) es una convención. No prohíbe técnicamente el acceso, pero es una señal clara para otros programadores: "¡No toques esto directamente!".

Pilar 2: Abstracción (Ocultando la Complejidad)

No necesitas ser un mecánico para conducir. Solo necesitas conocer el volante y los pedales.

Una interfaz simple para una acción compleja:

class Coche:
    def __init__(self):
        # El coche está compuesto de otros objetos complejos
        self._motor = Motor()
        self._inyeccion = SistemaInyeccion()
        self._transmision = Transmision()
        self._velocidad = 0

    # Este es el único método que el conductor necesita conocer
    def acelerar(self):
        print("Pisas el acelerador...")
        # ¡Mucha complejidad oculta aquí!
        self._motor.aumentar_rpm()
        self._inyeccion.inyectar_combustible()
        self._transmision.cambiar_marcha()
        self._velocidad += 10
        print("El coche avanza más rápido.")

# --- Desde fuera, todo es muy simple ---

mi_coche = Coche()
mi_coche.acelerar() # <-- ¡Solo llamamos a un método!

Ocultando la Complejidad

Este es el principio de Abstracción. Ofrecemos una interfaz de alto nivel (`acelerar()`) que es muy fácil de usar y que esconde todos los detalles complejos de la implementación.

TÚ SOLO VES ESTO
mi_coche.acelerar()
PERO POR DENTRO OCURRE ESTO

_motor.aumentar_rpm()

_inyeccion.inyectar_combustible()

_transmision.cambiar_marcha()

Beneficio: Podemos cambiar todo el motor del coche, y mientras el método `acelerar()` siga funcionando, ¡el código que usa el coche no necesita cambiar en absoluto!

La Herencia y el Principio DRY

Don't Repeat Yourself (No te repitas) es una regla de oro en programación.

🚫 Código WET (Write Everything Twice)

Sin herencia, nos vemos obligados a copiar y pegar el código común, lo cual es peligroso y difícil de mantener.

class Coche:
    def __init__(self, marca):
        self.marca = marca
        self._velocidad = 0

    # --- CÓDIGO DUPLICADO ---
    def acelerar(self, cantidad):
        self._velocidad += cantidad
    def frenar(self, cantidad):
        self._velocidad -= cantidad
    # -----------------------

class Camion:
    def __init__(self, marca):
        self.marca = marca
        self._velocidad = 0

    # --- CÓDIGO DUPLICADO ---
    def acelerar(self, cantidad):
        self._velocidad += cantidad
    def frenar(self, cantidad):
        self._velocidad -= cantidad
    # -----------------------

✅ Código DRY (Don't Repeat Yourself)

Con herencia, definimos el código común una sola vez en la clase padre. ¡Mucho más limpio y seguro!

# El código común vive aquí, en un solo lugar
class Vehiculo:
    def __init__(self, marca):
        self.marca = marca
        self._velocidad = 0

    def acelerar(self, cantidad):
        self._velocidad += cantidad
    def frenar(self, cantidad):
        self._velocidad -= cantidad

# Las clases hijas heredan el código, no lo repiten
class Coche(Vehiculo):
    def invitar_a_ligue(self):
        # ...
        pass

class Camion(Vehiculo):
    def adelantar_en_autopista(self):
        # ...
        pass

Ventaja Clave: Si encontramos un error en `acelerar()`, con el método DRY solo tenemos que arreglarlo en un sitio (`Vehiculo`), y se arreglará automáticamente en `Coche`, `Camion` y cualquier otro vehículo que creemos.

Herencia en Código: La Sintaxis

Así le decimos a Python que una clase "es un tipo de" otra.

class Vehiculo:
    def __init__(self, marca):
        print(f"Construyendo un Vehiculo de marca {marca}")
        self.marca = marca
        self._velocidad = 0

    def acelerar(self, cantidad):
        self._velocidad += cantidad
        print(f"El {self.marca} acelera. Velocidad: {self._velocidad} km/h")

# 1. Indicamos la clase padre entre paréntesis
class Coche(Vehiculo):
    def __init__(self, marca):
        print("Construyendo un Coche...")
        # 2. Llamamos al constructor del padre para que haga su parte
        super().__init__(marca)
        # 3. Añadimos los atributos específicos del hijo
        self.amor_acumulado = 100

    def invitar_a_ligue(self):
        print(f"El {self.marca} se detiene. '¿Te llevo a casa? 😏'")

# --- ¡A usarlo! ---
mi_coche = Coche("Ferrari")
# SALIDA: Construyendo un Coche...
# SALIDA: Construyendo un Vehiculo de marca Ferrari

# Podemos usar un método HEREDADO de Vehiculo
mi_coche.acelerar(120)
# SALIDA: El Ferrari acelera. Velocidad: 120 km/h

# Y podemos usar un método PROPIO de Coche
mi_coche.invitar_a_ligue()
# SALIDA: El Ferrari se detiene. '¿Te llevo a casa? 😏'

1. `class Coche(Vehiculo):`

Esta es la sintaxis clave. Al poner `(Vehiculo)` después del nombre, le decimos a Python que `Coche` es una clase hija que hereda de `Vehiculo`.

2. `super().__init__(marca)`

Esta línea es CRUCIAL. `super()` se refiere a la clase padre (`Vehiculo`). Con esta llamada, le estamos diciendo: "Ejecuta el constructor del padre para que inicialice los atributos `marca` y `_velocidad`."

3. `self.amor_acumulado = 100`

Una vez que el padre ha hecho su trabajo, la clase hija puede añadir sus propios atributos, que no existen en el padre.

Resultado: El objeto `mi_coche` tiene acceso a TODOS los métodos y atributos, tanto los suyos como los de su padre `Vehiculo`.

Polimorfismo en Código

Así es como un bucle simple puede controlar una orquesta de objetos diferentes.

class Vehiculo:
    def hacer_sonido(self):
        # Un sonido genérico para cualquier vehículo
        return "Sonido de vehículo..."

# Cada clase hija SOBREESCRIBE el método del padre
class Coche(Vehiculo):
    def hacer_sonido(self):
        return "¡Beep, beep!"

class Camion(Vehiculo):
    def hacer_sonido(self):
        return "¡MOOOOC, MOOOOC!"

class Bicicleta(Vehiculo):
    def hacer_sonido(self):
        return "¡Ring, ring!"

# --- La magia del polimorfismo ---
mi_garage = [ Coche(), Camion(), Bicicleta() ]

# Este bucle no necesita saber qué tipo de vehículo es cada uno
for vehiculo in mi_garage:
    # Simplemente confía en que cada objeto tiene un método .hacer_sonido()
    print(vehiculo.hacer_sonido())

# SALIDA:
# ¡Beep, beep!
# ¡MOOOOC, MOOOOC!
# ¡Ring, ring!

Sobrescritura de Métodos

Cuando una clase hija define un método con el mismo nombre que su padre, lo "sobrescribe". Esto le permite dar su propia implementación específica, como los diferentes sonidos.

Código Flexible y Limpio

El bucle `for` es increíblemente simple. No necesita `if` para preguntar "si el vehículo es un Coche, haz beep... si es un Camión, haz MOOC...". Esto es polimorfismo en acción.

El Gran Beneficio: Si mañana creamos una nueva clase `Moto(Vehiculo)`, ¡solo tenemos que añadirla a la lista `mi_garage` y el bucle funcionará sin tener que modificarlo en absoluto!

Ejercicio Práctico (Parte 1): El Mini-RPG

¡Manos a la obra! Vamos a aplicar todo lo aprendido creando la base de un juego de rol.

La Clase Base `Personaje`:

class Personaje:
    def __init__(self, nombre, vida, puntos_ataque):
        self.nombre = nombre
        self.vida = vida
        self.puntos_ataque = puntos_ataque
        print(f"¡Ha nacido {self.nombre}!")

    def atacar(self, enemigo):
        print(f"⚔️ ¡{self.nombre} ataca a {enemigo.nombre}!")
        enemigo.vida -= self.puntos_ataque
        
        if enemigo.vida <= 0:
            print(f"☠️ {enemigo.nombre} ha sido derrotado.")
        else:
            print(f"{enemigo.nombre} ahora tiene {enemigo.vida} puntos de vida.")

    def mostrar_estado(self):
        print(f"❤️ {self.nombre}: {self.vida} HP")

# --- Creando y usando nuestros primeros personajes ---
heroe = Personaje("Aragorn", 100, 15)
# SALIDA: ¡Ha nacido Aragorn!

orco = Personaje("Uglúk", 80, 10)
# SALIDA: ¡Ha nacido Uglúk!

heroe.atacar(orco)
# SALIDA: ⚔️ ¡Aragorn ataca a Uglúk!

Nuestra Plantilla

La clase `Personaje` es nuestro "plano" para cualquier entidad que pueda luchar en nuestro juego. Define los atributos y métodos comunes a todos.

El Método `atacar`

Fíjate cómo el método recibe `enemigo` como parámetro. Este `enemigo` es, en sí mismo, ¡otro objeto de tipo `Personaje`! Esto demuestra cómo los objetos pueden interactuar entre sí.

Ejercicio Práctico (Parte 2): Herencia en el RPG

¡Especialicemos a nuestros personajes! Un Guerrero no lucha igual que un Mago.

# Clase Padre (de la diapositiva anterior)
class Personaje:
    def __init__(self, nombre, vida, puntos_ataque):
        self.nombre = nombre; self.vida = vida; self.puntos_ataque = puntos_ataque
    def atacar(self, enemigo):
        print(f"⚔️ ¡{self.nombre} ataca a {enemigo.nombre}!")
        enemigo.vida -= self.puntos_ataque
    def mostrar_estado(self): print(f"❤️ {self.nombre}: {self.vida} HP")

# --- Clases Hijas ---
class Guerrero(Personaje):
    def __init__(self, nombre, vida, puntos_ataque):
        super().__init__(nombre, vida, puntos_ataque)
        self.furia = 0
    
    # Sobrescribimos el método atacar
    def atacar(self, enemigo):
        # El ataque del guerrero es más fuerte
        daño = self.puntos_ataque + self.furia
        print(f"🔥 ¡{self.nombre} ataca con furia ({daño} pts)!")
        enemigo.vida -= daño
        self.furia += 5 # Acumula furia con cada ataque

class Mago(Personaje):
    def __init__(self, nombre, vida, puntos_ataque):
        super().__init__(nombre, vida, puntos_ataque)
        self.mana = 100
    
    # Añadimos un método nuevo
    def lanzar_hechizo(self, enemigo):
        if self.mana >= 20:
            print(f"✨ ¡{self.nombre} lanza una bola de fuego!")
            enemigo.vida -= 30 # Daño mágico fijo
            self.mana -= 20
        else:
            print("¡No hay suficiente maná!")

# --- Combate ---
gimli = Guerrero("Gimli", 120, 12)
gandalf = Mago("Gandalf", 80, 8)
orco = Personaje("Lurtz", 90, 10)

gimli.atacar(orco)
gandalf.lanzar_hechizo(orco)
gimli.atacar(orco) # El segundo ataque es más fuerte

Guerrero

Hereda de `Personaje` pero sobrescribe `atacar` para añadir la mecánica de `furia`. Su ataque se vuelve más fuerte cada vez que lo usa.

Mago

También hereda de `Personaje`, pero en lugar de sobrescribir, añade un nuevo método, `lanzar_hechizo`, con su propia lógica de `mana`.

Polimorfismo Implícito

Aunque `gimli` es un `Guerrero`, todavía podemos pasarlo como el `enemigo` al método `atacar` de otro personaje, porque "es un tipo de" `Personaje`.

¡Hemos combinado Herencia y Polimorfismo para crear un sistema de combate flexible y fácil de extender!

Ejercicio Práctico (Parte 3): Polimorfismo en el RPG

¡La culminación! Tratemos a todos los personajes por igual, sin importar su clase.

# ... (Clases Personaje, Guerrero, Mago de la slide anterior)
# Añadimos un método "presentarse" a cada clase

class Personaje:
    # ... (init, atacar, etc.)
    def presentarse(self):
        print(f"Soy {self.nombre}, un aventurero.")

class Guerrero(Personaje):
    # ... (init, atacar con furia, etc.)
    def presentarse(self):
        print(f"Soy {self.nombre}, ¡un guerrero con {self.furia} de furia!")

class Mago(Personaje):
    # ... (init, lanzar_hechizo, etc.)
    def presentarse(self):
        print(f"Soy {self.nombre}, un mago con {self.mana} de maná.")


# --- La magia del Polimorfismo ---
grupo = [
    Guerrero("Gimli", 120, 12),
    Mago("Gandalf", 80, 8),
    Personaje("Bilbo", 50, 5) # Un personaje genérico
]

print("--- Presentación del grupo ---")
for miembro in grupo:
    # ¡No necesitamos saber qué tipo de personaje es!
    # Solo llamamos al método que todos comparten.
    miembro.presentarse()

# SALIDA:
# --- Presentación del grupo ---
# Soy Gimli, ¡un guerrero con 0 de furia!
# Soy Gandalf, un mago con 100 de maná.
# Soy Bilbo, un aventurero.

Método Común, Distinta Implementación

Todas las clases (`Personaje`, `Guerrero`, `Mago`) tienen un método llamado `presentarse`. Cada una lo implementa a su manera para mostrar su estado particular.

Un Bucle para Gobernarlos a Todos

El bucle `for` trata a cada `miembro` de la lista como un `Personaje`. No le importa si en realidad es un `Guerrero` o un `Mago`. Simplemente llama a `miembro.presentarse()` y confía en que el objeto sabrá qué hacer.

Escalabilidad Futura

Si mañana creamos la clase `Arquero(Personaje)`, solo tenemos que añadir un arquero a la lista `grupo`, y este código seguirá funcionando sin cambios. ¡Ese es el poder de la OOP!
