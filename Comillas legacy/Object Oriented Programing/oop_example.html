Paso 1: El Calculador de Pricing

Todo sistema necesita un "precio justo". Lo modelamos con un objeto que encapsula esta lógica.

La Clase `CalculadorPricing`:

class CalculadorPricing:
    """
    Un objeto simple cuya única responsabilidad es
    calcular el precio medio de un activo.
    """
    def calcular_mid_price(self, best_bid, best_ask):
        """Calcula el punto medio entre la mejor compra y venta."""
        if best_bid is None or best_ask is None:
            return None # No podemos calcular sin datos
        
        mid_price = (best_bid + best_ask) / 2
        return mid_price

# --- Uso del objeto ---
# 1. Obtenemos datos del mercado (simulados)
lob_best_bid = 99.98
lob_best_ask = 100.02

# 2. Creamos una instancia de nuestro calculador
pricing_calculator = CalculadorPricing()

# 3. Le pedimos que haga su trabajo
precio_justo = pricing_calculator.calcular_mid_price(
    lob_best_bid, 
    lob_best_ask
)

print(f"El precio justo (mid-price) es: {precio_justo}")
# SALIDA: El precio justo (mid-price) es: 100.0

Diagrama de Flujo

Datos del LOB

Best Bid: 99.98
Best Ask: 100.02

Objeto: `CalculadorPricing`

.calcular_mid_price()

Resultado (Mid-Price): 100.0

Encapsulación: La lógica para calcular el precio está contenida en su propia clase. Si el modelo de pricing cambia (ej. a un VWAP), solo modificamos esta clase, ¡y el resto del sistema no se entera!

Paso 2: El Modelo de Quotes y Spread

Con el precio justo, ahora decidimos a qué precios queremos comprar (bid) y vender (ask).

La Clase `ModeloDeQuotes`:

class ModeloDeQuotes:
    """
    Calcula los precios de compra y venta aplicando
    un spread alrededor del precio medio.
    """
    def calcular_quotes(self, mid_price, spread_bps):
        """Calcula bid y ask a partir del mid-price."""
        if mid_price is None:
            return None, None
            
        spread = mid_price * (spread_bps / 10000) / 2
        
        # Nuestros precios para operar
        bid_quote = mid_price - spread
        ask_quote = mid_price + spread
        
        # Redondeamos a 2 decimales para que sean precios válidos
        return round(bid_quote, 2), round(ask_quote, 2)

# --- Uso del objeto ---
# 1. Usamos el mid-price del paso anterior
precio_justo = 100.0

# 2. Definimos nuestro spread (ej. 4 basis points)
spread_deseado_bps = 4

# 3. Creamos una instancia de nuestro modelo
quoting_model = ModeloDeQuotes()

# 4. Le pedimos que calcule nuestros precios
mi_bid, mi_ask = quoting_model.calcular_quotes(
    precio_justo, 
    spread_deseado_bps
)

print(f"Mi quote es: Bid={mi_bid} | Ask={mi_ask}")
# SALIDA: Mi quote es: Bid=99.98 | Ask=100.02

Diagrama de Flujo

Input

Mid-Price: 100.0
Spread: 4 bps

Objeto: `ModeloDeQuotes`

.calcular_quotes()

New Bid: 99.98
New Ask: 100.02

Paso 3: El Gestor de Órdenes y el LOB

Con nuestros precios, creamos órdenes y las enviamos al mercado (nuestro LOB simulado).

Poniéndolo todo junto:

# Clase para representar una orden
class Orden:
    def __init__(self, side, price, qty):
        self.side = side # "buy" o "sell"
        self.price = price
        self.qty = qty
    def __repr__(self): # Para que se imprima de forma legible
        return f"Orden({self.side}, P={self.price}, Q={self.qty})"

# Simulación de nuestro libro de órdenes
class LimitOrderBook:
    def __init__(self):
        self.ordenes = []
    def add_order(self, orden):
        print(f"📬 Enviando orden al LOB: {orden}")
        self.ordenes.append(orden)

# --- Flujo completo ---
# 1. Usamos los precios del paso anterior
mi_bid, mi_ask = 99.98, 100.02
cantidad = 10

# 2. Creamos los objetos Orden
orden_de_compra = Orden("buy", mi_bid, cantidad)
orden_de_venta = Orden("sell", mi_ask, cantidad)

# 3. Creamos una instancia de nuestro LOB
lob = LimitOrderBook()

# 4. Enviamos las órdenes
lob.add_order(orden_de_compra)
lob.add_order(orden_de_venta)

print("\nÓrdenes en el libro:", lob.ordenes)
# SALIDA: 📬 Enviando orden al LOB: Orden(buy, P=99.98, Q=10)
# SALIDA: 📬 Enviando orden al LOB: Orden(sell, P=100.02, Q=10)
# SALIDA: Órdenes en el libro: [Orden(buy, ...), Orden(sell, ...)]

La Orquesta de Objetos

El `GestorDeOrdenes` (nuestro script principal) actúa como el director de orquesta, coordinando los diferentes objetos, cada uno con su responsabilidad única:

`CalculadorPricing`: Sabe de precios.

`ModeloDeQuotes`: Sabe de spreads y quotes.

`Orden`: Sabe representar una orden.

`LimitOrderBook`: Sabe cómo gestionar un libro de órdenes.

¿Por Qué Es Mejor Así?

Separar las responsabilidades hace que el sistema sea:

  • Fácil de testear: Podemos probar el `CalculadorPricing` de forma aislada.
  • Fácil de modificar: Si queremos un spread dinámico, solo cambiamos `ModeloDeQuotes`.
  • Fácil de entender: Cada clase tiene un propósito claro y definido.

Paso 4: La Estrategia (El Orquestador)

Creamos un objeto "jefe" que contiene a nuestros calculadores. A esto se le llama Composición.

La Clase `TradingStrategy`:

class TradingStrategy:
    """
    Este objeto contiene otros objetos y orquesta el flujo.
    Un objeto "TIENE UN" CalculadorPricing.
    Un objeto "TIENE UN" ModeloDeQuotes.
    """
    def __init__(self, spread_bps):
        # La estrategia CREA y GUARDA sus herramientas
        self.pricing_model = CalculadorPricing()
        self.quoting_model = ModeloDeQuotes()
        self.spread_bps = spread_bps

    def generar_quotes(self, best_bid, best_ask):
        """
        Orquesta el flujo:
        1. Calcula el mid-price usando su herramienta.
        2. Calcula los quotes usando su otra herramienta.
        """
        mid_price = self.pricing_model.calcular_mid_price(
            best_bid, best_ask
        )
        
        bid, ask = self.quoting_model.calcular_quotes(
            mid_price, self.spread_bps
        )
        
        return bid, ask

# --- Uso ---
# Creamos una instancia de la estrategia completa
mi_estrategia = TradingStrategy(spread_bps=4)

# Le damos los datos del mercado y nos devuelve los quotes
mi_bid, mi_ask = mi_estrategia.generar_quotes(99.98, 100.02)

print(f"La estrategia ha generado: Bid={mi_bid} | Ask={mi_ask}")
# SALIDA: La estrategia ha generado: Bid=99.98 | Ask=100.02

Composición vs. Herencia

Herencia (ES UN)

Un `Coche` ES UN `Vehiculo`. Es una relación de tipo.

Composición (TIENE UN)

Una `TradingStrategy` TIENE UN `CalculadorPricing`. No es un tipo de calculador, sino que lo usa como herramienta.

Abstracción en acción: Desde fuera, solo necesitamos llamar a `mi_estrategia.generar_quotes()`. No nos importa cómo calcula el mid-price o el spread, toda esa complejidad está oculta (abstraída) dentro del objeto.

Paso 5: Los Datos Como Objetos

En lugar de pasar muchos argumentos, pasamos un solo objeto que contiene toda la información.

La Clase `DatosMercado`:

from dataclasses import dataclass

@dataclass
class DatosMercado:
    """Un objeto simple para agrupar datos de entrada."""
    best_bid: float
    best_ask: float
    timestamp: int

# --- Refactorizamos la estrategia para aceptar este objeto ---
class TradingStrategy:
    def __init__(self, spread_bps):
        self.pricing_model = CalculadorPricing()
        self.quoting_model = ModeloDeQuotes()
        self.spread_bps = spread_bps

    def generar_quotes(self, datos: DatosMercado):
        # Ahora el método es más limpio, solo recibe un objeto
        mid_price = self.pricing_model.calcular_mid_price(
            datos.best_bid, datos.best_ask
        )
        bid, ask = self.quoting_model.calcular_quotes(
            mid_price, self.spread_bps
        )
        return bid, ask

# --- Uso ---
# 1. Creamos un objeto con los datos del mercado
datos_actuales = DatosMercado(
    best_bid=99.98,
    best_ask=100.02,
    timestamp=1678886400
)

# 2. La llamada al método es mucho más limpia
mi_estrategia = TradingStrategy(spread_bps=4)
mi_bid, mi_ask = mi_estrategia.generar_quotes(datos_actuales)

print(f"Quote generado a partir de objeto de datos: Bid={mi_bid}")
# SALIDA: Quote generado a partir de objeto de datos: Bid=99.98

¿Por Qué Hacer Esto?

Antes (Frágil)

def generar_quotes(self, bid, ask, vol, timestamp, ...):

Si queremos añadir un nuevo dato (ej. `last_trade_price`), ¡tenemos que cambiar la definición del método en todos sitios!

Ahora (Robusto y Escalable)

def generar_quotes(self, datos: DatosMercado):

Si queremos añadir `last_trade_price`, solo lo añadimos a la clase `DatosMercado`. ¡El método `generar_quotes` no necesita cambiar!

Paso 6: El Flujo Avanzado y Completo

Así se ve la "orquesta" de objetos trabajando en armonía.

El Script Principal:

# ... (Definiciones de todas las clases anteriores)
# CalculadorPricing, ModeloDeQuotes, Orden, LimitOrderBook
# DatosMercado, TradingStrategy

# --- Punto de Entrada Principal (main) ---

# 1. Creamos nuestras herramientas y el mercado
mi_estrategia = TradingStrategy(spread_bps=4)
lob = LimitOrderBook()
cantidad_orden = 10

# 2. Llega un nuevo "tick" de datos del mercado
datos_tick_1 = DatosMercado(99.98, 100.02, 1)

# 3. La estrategia procesa los datos y genera quotes
bid, ask = mi_estrategia.generar_quotes(datos_tick_1)

# 4. Creamos objetos Orden con esos quotes
orden_compra = Orden("buy", bid, cantidad_orden)
orden_venta = Orden("sell", ask, cantidad_orden)

# 5. Enviamos las órdenes al LOB
lob.add_order(orden_compra)
lob.add_order(orden_venta)

# --- Llega otro tick ---
datos_tick_2 = DatosMercado(100.01, 100.05, 2)
bid, ask = mi_estrategia.generar_quotes(datos_tick_2)
# ... y el ciclo se repite ...

El Poder de la Abstracción

Observa lo limpio y legible que es el script principal. Es como leer un resumen de alto nivel de lo que hace el sistema, porque cada objeto esconde su propia complejidad.

  • 1. Crea los actores: `Estrategia`, `Mercado`.
  • 2. Recibe `Datos`.
  • 3. La `Estrategia` genera `Quotes`.
  • 4. Crea `Órdenes`.
  • 5. Envía al `Mercado`.

Un Sistema Preparado para Crecer

Este diseño nos permite:

  • Añadir Estrategias: Crear una `TrendingStrategy(TradingStrategy)` que herede y cambie el cálculo del spread.
  • Cambiar el Mercado: Reemplazar `LimitOrderBook` por una conexión a un exchange real (`BinanceGateway`) sin cambiar la estrategia.
  • Trabajar en Equipo: Un desarrollador puede mejorar `CalculadorPricing` mientras otro trabaja en el `LimitOrderBook`, ¡sin pisarse!
